ExplorestarsLite User Manual

Explore Scientific PMC-Eight™ Telescope Mount Control — Android App

Version 1.2.2 • March 2026

1. Overview

ExplorestarsLite is a standalone Android app that controls Explore Scientific PMC-Eight™ telescope mounts directly from your phone via WiFi. No PC, laptop, or internet connection is required.

The app communicates with the PMC-Eight through the ESP32 WiFi module built into the mount controller. Your phone sends commands over HTTP to the ESP32, which forwards them to the PMC-Eight's Propeller processor.

Architecture: Phone → WiFi → ESP32 HTTP bridge (port 80) → TCP loopback → PMC-Eight Propeller firmware

Key features:

  • 10,900+ object catalog with altitude-based visibility coloring
  • GoTo, Sync, Tracking (Sidereal, Solar, Lunar)
  • Spiral search with video-remote-style transport controls
  • Solar system targets: planets, Moon, Sun, and minor planets (Ceres, Vesta, Pallas, Juno)
  • Meridian flip with auto-flip option
  • 12 astronomy calculators
  • Night vision mode for dark adaptation
  • Firmware configuration reader/writer
  • Works alongside Alpaca/ASCOM and planetarium software (e.g., Cartes du Ciel)

2. Installation


Tap QR or here to download latest APK
  1. Scan the QR code above or tap the download link (switch to the explorestars-lite branch).
  2. Tap the APK file, then tap the three dots icon and select Download.
  3. Tap the downloaded file to install. Allow "Install from unknown sources" if prompted.
  4. If Google Play Protect warns you, tap "Install anyway".
Tip: On Android, scan QR codes using the Google app (tap the camera icon in the search bar). The built-in Camera app may not scan QR codes on all devices.

3. Quick Start

  1. Power on your PMC-Eight mount and wait for WiFi to start (~10 seconds).
  2. Connect your phone to the mount's WiFi network (default SSID: PMC-Eight, default IP: 192.168.47.1).
  3. Open ExplorestarsLite.
  4. On the Setup page, verify the IP address and tap CONNECT.
  5. Enter your latitude and longitude (required for accurate pointing).
  6. Go to the Target page, search for an object, and tap GOTO.

5. Setup Page

5.1 Connection

Enter the PMC-Eight's IP address and select your mount model, then tap CONNECT.

FieldDefaultDescription
PMC-Eight IP Address192.168.47.1IP of the PMC-Eight WiFi interface. Use the default for AP mode, or enter your LAN IP for home network use.
Mount ModeliExos-100Determines encoder counts used for position calculations. Options: Losmandy G-11, iExos-100, EXOS-2.
Important: Selecting the wrong mount model will cause incorrect position calculations and pointing errors.

5.2 Site Location & Time

Enter your observing site coordinates. These are essential for accurate Alt/Az calculations, LST, and object visibility.

  • Latitude — positive = North. Enter as decimal degrees (32.881) or DMS (32:52:52).
  • Longitude — positive = East. Enter as decimal degrees (-97.123) or DMS (-97:07:23).

Local time and UTC are displayed for reference. Time comes from your phone's clock.

Tip: All settings (IP, mount model, location, TFOV, step fraction) are saved automatically and persist between sessions.

5.3 Tools

  • Firmware Config — read/write PMC-Eight firmware settings (requires connection). See Section 9.
  • Calculators — 12 astronomy calculators. See Section 8.
  • Polar Alignment — daylight polar alignment using your phone's compass and accelerometer. See Section 10.

6. Target Page

Use this page to find celestial objects and send the mount to them.

6.1 Catalog Search

Type in the search field to filter the 10,900+ object catalog. Results appear in a scrollable list showing the first 50 matches.

  • Green text — object is above 10° altitude (good for observing)
  • Gray text — object is below 10° altitude

Tap any result to select it as your target. The search icon reopens the last search results.

Coordinate system: All catalog positions are stored in J2000 and automatically precessed to JNOW (current equinox) for pointing.

6.2 Nearby Search

Tap Nearby to find catalog objects within a configurable radius of the mount's current position. Useful for identifying what you're pointed at. Adjust the radius (1–90°) with the spinner.

Auto-detect on connect: When you connect and the mount is not parked, the app automatically searches for the nearest catalog object and selects it.

6.3 Solar System

Select a solar system body from the dropdown: Sun, Moon, planets (Mercury through Neptune), Pluto, and minor planets (Ceres, Vesta, Pallas, Juno).

  • Blue text — above the horizon
  • Gray text — below the horizon

Positions are computed offline using Meeus algorithms — no internet needed.

The tracking rate is set automatically: Solar rate for the Sun, Lunar rate for the Moon, Sidereal for everything else.

6.4 Custom Coordinates & Saved Targets

Tap the pencil icon next to the target heading to open the custom coordinate dialog. Enter RA and Dec manually in hours/degrees, minutes, seconds format.

  • Choose J2000 if entering catalog coordinates (will be precessed to JNOW).
  • Choose JNOW if entering current-epoch coordinates.
  • Proper motion — optionally enter PM RA·cos(δ) and PM Dec in mas/yr for high-proper-motion stars.
  • Name the target to save it permanently. Named targets are stored in the app and appear in the Saved Targets list on the Target page whenever the search field is empty.

Managing Saved Targets

To view your saved targets, clear the search field — the saved targets list appears automatically in the results area. Each entry has:

  • Tap — select the target for GoTo/Sync.
  • Edit (pencil icon) — re-opens the custom coordinate dialog with the target's data pre-filled. Save with the same name to update it.
  • Delete (trash icon) — permanently removes the target from saved storage.

Saved targets also appear in catalog search results when their name matches the search text. They are persisted across sessions and app restarts.

6.5 GoTo & Sync

  • GOTO — slew the mount to the selected target. Tracking starts automatically after the slew completes.
  • SYNC — tell the mount that it is currently pointed at the selected target (corrects pointing model).
Cross-meridian guard: Sync is blocked when the mount's pier side does not match the target's expected pier side. This prevents sync errors that could cause pointing failures.

6.6 Alt/Az GoTo

Enter altitude and azimuth in degrees and tap GOTO Alt/Az to slew the mount to those coordinates. Useful for terrestrial targets or manual positioning.

7. Control Page

7.1 Tracking

Tap ON to start sidereal tracking (or the rate set by your target selection). Tap OFF to stop. The button is green when tracking is active, red when stopped.

7.2 Meridian Flip

German equatorial mounts must flip when tracking past the meridian to avoid hardware collision.

  • Pier side is displayed as "Pier: W" or "Pier: E" (orange text).
  • Auto flip toggle — automatically flips when the mount tracks more than 2 minutes past the meridian.
  • FLIP button — manually trigger a meridian flip. Enabled when connected, a target is selected, and HA ≥ -2 minutes.

Flip sequence: stop tracking → compute opposite pier motor angles → slew → wait for slew to complete → restart tracking.

7.3 Spiral Search

Spiral search slews the mount in an expanding spiral pattern around your target, useful for centering objects that are near but not in the eyepiece.

Setup

  • TFOV — True Field of View in degrees. Set this using the Eyepiece TFOV or Imaging FOV calculator.
  • Fraction — overlap between spiral steps (0.1 to 1.0). Lower values give more overlap.
  • Step size = TFOV × Fraction (displayed in green).

Transport Controls

Controls are styled like a video remote:

ButtonAction
⏮Skip to previous ring
◁◁Step backward one position
▶ / ⏸Play (auto-advance) / Pause
▷▷Step forward one position
⏭Skip to next ring

A Stop button (red) appears during active search to cancel.

Manual stepping auto-pauses a running search. The search center initializes from the selected target.

7.4 Thumb Pad & Direction

The thumb pad is a large circular touch area for manual centering. Touch and drag in any direction to slew; release to stop. Only one axis moves at a time (the dominant direction).

  • Deadband — small area in the center where no motion occurs.
  • N/S — DEC axis motion.
  • E/W — RA axis motion, symmetric relative to tracking (see Slew Rate below).
  • Release — DEC stops, RA returns to tracking rate (sidereal, solar, or lunar as appropriate).

The Direction section also shows:

  • Target Alt/Az (cyan) — where the selected target is in the sky.
  • Mount Alt/Az/HA (orange/blue) — where the mount is currently pointing.

Below the thumb pad, three action buttons:

  • GOTO — slew to the selected target.
  • HOME — slew to the home/park position.
  • SYNC — sync the mount to the selected target.

7.5 Slew Rate

A discrete-stop slider selects the slew rate used by the thumb pad. The first two positions are configurable; the rest are fixed rates.

Rate Stops

PositionModeDescription
1Fine1–4× sidereal. Tap the Fine label to configure.
2Coarse0.1–0.4°/s. Tap the Coarse label to configure.
3–12Fixed0.02, 0.05, 0.075, 0.1, 0.2, 0.5, 1.0, 1.4, 1.5, 2.0 °/s

Symmetric RA Motion (Fine & Coarse)

In Fine and Coarse modes, RA slewing is symmetric relative to tracking. Moving east (against sidereal) and west (with sidereal) produces the same apparent speed in the eyepiece. This is achieved by adding or subtracting the selected rate from the sidereal tracking rate.

Tip: Fine 1× is ideal for precise centering at the eyepiece. The object drifts at sidereal rate in one direction and stops in the other. Fine 3–4× is good for faster centering. Coarse is for larger corrections.

10. Daylight Polar Alignment

Align your mount's polar axis to the celestial pole using your phone's compass and accelerometer — no stars needed. Access from Setup → Polar Alignment.

How It Works

The display shows two crosshairs:

  • Red + — the celestial pole (NCP or SCP, based on your latitude)
  • Green + — where your phone is pointing

Three concentric circles show Telrad-style scale rings at 0.5°, 1°, and 1.5°. A connecting line shows the direction and distance to the pole.

Procedure

  1. Verify your latitude and longitude are set correctly on the Setup page.
  2. Enter your local magnetic declination (the difference between true north and magnetic north). Find your value at NOAA's calculator. East declination is positive, west is negative.
  3. Calibrate the compass by moving the phone slowly in a figure-8 pattern.
  4. Place the phone flat on the mount's RA axis (or on a flat surface aligned with the axis).
  5. Tap Start Alignment.
  6. Adjust the mount's altitude (up/down) and azimuth (left/right) until the green crosshair overlaps the red crosshair.
Phone requirements: Your phone must have a magnetometer (compass) and accelerometer. Most iPhones have both. Some budget Android phones lack a magnetometer — in that case, only altitude adjustment will work (azimuth requires the compass).
Tip: On iPhone, the polar alignment page must be accessed over HTTPS for compass access. The Android APK does not have this restriction.

11. Night Vision Mode

Tap the moon icon in the app bar to toggle night vision mode. The screen shifts to deep red tones to preserve your dark adaptation while observing.

The mode uses a CSS filter that converts all colors to red and reduces brightness. Tap the moon icon again to return to normal mode.

12. Networking

12.1 AP Mode (Field Use)

The default configuration for field use. The PMC-Eight creates its own WiFi network.

  1. Power on the PMC-Eight.
  2. On your phone, connect to the mount's WiFi network (SSID: PMC-Eight XXXX, where XXXX is a unique 4-digit code for your unit). The password is PMC-Eight.
  3. In the app, use the default IP: 192.168.47.1.

Multiple devices may connect simultaneously — each will have full control and can display mount status. The mount responds to any command regardless of source, so users must coordinate to avoid sending conflicting commands.

12.2 LAN Mode (Home Use)

For home use, you can configure the PMC-Eight to join your home WiFi network. This allows the mount, your phone, and your PC to all be on the same network simultaneously.

  1. Configure the PMC-Eight to join your home WiFi (see OTA quick start guide).
  2. Find the PMC-Eight's IP address on your LAN (check your router's DHCP table).
  3. In the app, enter that LAN IP address.
Multi-client: In LAN mode, ExplorestarsLite on your phone can run alongside Alpaca/ASCOM and planetarium software (e.g., Cartes du Ciel) on your PC, all controlling the same mount.

12.3 Alpaca Server Compatibility

ExplorestarsLite works seamlessly alongside the PMC-Eight ASCOM Alpaca telescope driver. The Alpaca server may connect to the PMC-Eight either serially (USB) or through WiFi, and may be configured to use either TCP or UDP — the PMC-Eight supports both simultaneously without specific configuration. The app and Alpaca can communicate with the mount at the same time.

When running together:

  • Either can control the mount — GoTo, tracking, slewing, and sync commands from ExplorestarsLite or any ASCOM Alpaca Server connected planetarium software (e.g., Cartes du Ciel, Stellarium, SkySafari) are accepted by the mount.
  • Real-time status — ExplorestarsLite displays the mount's current position, tracking state, and slew status regardless of which client issued the command. If your planetarium software starts a GoTo, the app shows "Slewing" and updates the position in real time.
  • Shared tracking — If Alpaca starts or stops tracking, ExplorestarsLite detects the rate change within a few seconds and updates its display.
  • Manual centering — Use the app's thumb pad for fine centering while your planetarium software maintains the target reference. This is especially useful for visual observing sessions where the PC handles planning and the phone handles eyepiece work.

No special setup is required. Simply connect both the app and Alpaca to the same PMC-Eight, either in AP mode or on your home LAN.

14. Troubleshooting

ProblemSolution
App says "Failed" on connectVerify your phone is connected to the mount's WiFi network. Check the IP address. Ensure the PMC-Eight is powered on and WiFi has started (~10 seconds).
Connected but no position updatesCheck that the correct mount model is selected. Try disconnecting and reconnecting.
GoTo points to wrong locationVerify your latitude and longitude are correct. Check that you selected the correct mount model. Perform a sync on a known object first.
Sync is disabled / blockedThe cross-meridian guard blocks sync when pier sides don't match. Ensure you're on the correct side of the meridian for the target.
Spiral search step size is 0Set the TFOV using the Eyepiece TFOV or Imaging FOV calculator first.
Night vision mode too darkThis is by design for dark adaptation. Your eyes will adjust after a few minutes.
App installed but won't openEnsure you're running Android 7.0 (API 24) or later.
Settings lost after reinstallSettings are stored in the app's local storage. Reinstalling or clearing app data resets them. Re-enter your location and mount IP.
