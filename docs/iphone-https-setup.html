iPhone HTTPS Certificate Setup

ExplorestarsLite uses HTTPS for features that require a secure browser context, such as compass and accelerometer access for polar alignment. A one-time certificate install is needed on your iPhone.

You only need to do this once. The certificate stays installed until you remove it or it expires (10 years).

What You Need

  • Your iPhone connected to the same WiFi network as the PC running the server
  • The PC's IP address (e.g., 192.168.0.36)
  • The ExplorestarsLite server running on the PC (HTTP on port 5257)

Step 1 — Download the Profile

On your iPhone, open Safari (not Chrome — it must be Safari) and go to:

http://<computer-ip>:5257/caddy-trust.mobileconfig

Replace <computer-ip> with your PC's actual IP address. For example:

http://192.168.0.36:5257/caddy-trust.mobileconfig

Safari will show: "This website is trying to download a configuration profile. Do you want to allow this?"

Tap Allow.

Step 2 — Install the Profile

  1. Open the Settings app
  2. You should see a banner near the top: "Profile Downloaded" — tap it
    (If the banner has disappeared: go to Settings → General → VPN & Device Management)
  3. You'll see "ExplorestarsLite HTTPS" — tap it
  4. Tap Install (top right corner)
  5. Enter your iPhone passcode if prompted
  6. Tap Install again on the confirmation screen
  7. Tap Done

Step 3 — Trust the Certificate

Important: Installing the profile is not enough. You must also explicitly trust the certificate. This is a separate step that many people miss.
  1. Open Settings → General → About
  2. Scroll all the way to the very bottom of the About page
  3. Tap Certificate Trust Settings
  4. Find Caddy Local Authority - 2025 ECC Root
  5. Toggle the switch ON
  6. Tap Continue on the warning dialog

Step 4 — Verify

Open Safari and go to:

https://<computer-ip>:5256

For example:

https://192.168.0.36:5256

The ExplorestarsLite app should load without any certificate warnings. If you see the app's splash screen and Setup page, you're done!

Troubleshooting

"Cannot connect to server"

  • Make sure the server is running on the PC
  • Make sure iPhone and PC are on the same WiFi network
  • Try the HTTP version first: http://<computer-ip>:5257 — if that doesn't work, it's a network issue, not a certificate issue

"This Connection Is Not Private" or certificate warning on :5256

  • You skipped Step 3 — you installed the profile but did not trust the certificate. Go to Settings → General → About → Certificate Trust Settings and toggle it on.
  • If the certificate name does not match (e.g., year is different), the server generated a new certificate. Remove the old profile and repeat from Step 1.

Certificate not appearing in Trust Settings

  • Restart your iPhone and check again
  • Make sure Step 2 completed fully (profile shows as "Installed" under VPN & Device Management)

Profile download fails or nothing happens

  • Must use Safari — Chrome and other browsers cannot install profiles on iOS
  • If Safari asks to allow the download and you tap Cancel, just go to the URL again and tap Allow
  • If the caddy-trust.mobileconfig file does not exist, stop the server, delete the file wwwroot/caddy-trust.mobileconfig if it exists, and restart — the server will regenerate it on startup

Accidentally denied GPS / location permission

  • If you added ExplorestarsLite to your home screen: go to Settings → ExplorestarsLite → Location and change to While Using
  • If using Safari directly: go to Settings → Safari → Location and change to Ask or While Using

Compass not working even on HTTPS

  • Make sure you are on https://:5256 (not http://:5257) — compass requires HTTPS
  • The first time, the app will ask for compass/motion permission — tap Allow
  • If you denied it: go to Settings → Safari → Motion & Orientation Access and toggle it on. Then reload the page.

Still getting certificate warnings after doing everything

  • Remove the old profile: Settings → General → VPN & Device Management → tap the profile → Remove Profile
  • Restart your iPhone
  • Clear Safari cache: Settings → Safari → Clear History and Website Data
  • Repeat from Step 1

Do I need HTTPS?

No — HTTPS is optional. The app works fully on HTTP (:5257) for GoTo, tracking, star alignment, and all other features. HTTPS (:5256) is only needed on iPhone for polar alignment (compass) and GPS location, because Apple requires HTTPS for sensor access. If you do not need polar alignment or GPS, skip this entire guide.
